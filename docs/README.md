# Documentação do ShipLua

A documentação canônica do projeto permanece em português do Brasil. Cada
documento em português possui uma versão inglesa pareada no mesmo diretório,
com o sufixo `.en.md` antes da extensão.

Exemplos:

- `PLAN.md` → `PLAN.en.md`;
- `docs/api/schema-contract.md` → `docs/api/schema-contract.en.md`;
- `rfcs/0001-shiplua-architecture.md` → `rfcs/0001-shiplua-architecture.en.md`.

As versões inglesas preservam código, comandos, caminhos, identificadores e
contratos técnicos. Em caso de divergência, consulte o documento em português,
que continua sendo a fonte canônica do registro do projeto.
